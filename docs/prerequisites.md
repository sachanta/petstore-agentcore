# Prerequisites & Deployment Plan

## Overview

This project deploys a Virtual Pet Store AI agent on AWS Bedrock AgentCore. It consists of two layers of infrastructure:

- **Layer 1 (YAML / CloudFormation):** RAG infrastructure тАФ OpenSearch Serverless, Bedrock Knowledge Bases, Guardrail, and supporting Lambda functions.
- **Layer 2 (Notebook):** Agent runtime тАФ Docker image built via CodeBuild, pushed to ECR, and deployed as a Bedrock AgentCore Runtime.

- Create a separate S3 bucket for CodeBuild source uploads (e.g. `petstore-codebuild-<account-id>`)
- Update `CodeBucketForAutomationARN` in the notebook Step 2 parameters

### 5. Lambda Functions тАФ Enterprise System Backends

The agent calls two Lambda functions as its "system of record" backends. These must exist before running the agent:

- **Inventory Management Lambda** (`SYSTEM_FUNCTION_1_NAME`): handles `getInventory` requests
- **User Management Lambda** (`SYSTEM_FUNCTION_2_NAME`): handles `getUserById` and `getUserByEmail` requests

See the function contracts in `pet_store_agent/inventory_management.py` and `pet_store_agent/user_management.py`.

## Build Plan

Execute steps in this exact order:

```
Step 1  Create IAM SolutionAccessRole
Step 2  Create S3 bucket for knowledge data
Step 3  Upload pet-store-products/ files to S3
Step 4  Create S3 bucket for code staging
Step 5  Deploy Lambda functions (inventory + user management backends)
Step 6  Deploy CloudFormation stack
          aws cloudformation deploy \
            --template-file pet_store_knowledge_bases.yaml \
            --stack-name pet-store-knowledge-bases \
            --capabilities CAPABILITY_IAM CAPABILITY_AUTO_EXPAND \
            --parameter-overrides \
              SolutionAccessRoleArn=<role-arn> \
              DataBucketForKnowledgeArn=<bucket-arn>
Step 7  Wait for stack CREATE_COMPLETE (~5-10 mins due to AOSS bootstrapping)
Step 8  Trigger S3 knowledge base ingestion sync (not auto-triggered)
          aws bedrock-agent start-ingestion-job \
            --knowledge-base-id <ProductInfo1stKnowledgeBaseId> \
            --data-source-id <S3DataSourceId>
Step 9  Wait for ingestion job to complete
Step 10 Update notebook Step 2 parameters with stack output values
Step 11 Run notebook cells in order (Steps 0тАУ8)
```

Stack outputs to capture after Step 6:
```
ProductInfo1stKnowledgeBaseId  тЖТ KNOWLEDGE_BASE_1_ID in notebook
PetCare2ndKnowledgeBaseId      тЖТ KNOWLEDGE_BASE_2_ID in notebook
PetStoreGuardrailId            тЖТ reference only
PetStoreGuardrailVersion       тЖТ reference only
PetStoreCollectionARN          тЖТ reference only
```

## Destroy Plan

Execute steps in this exact order (reverse of build):

```
Step 1  Delete AgentCore Runtime (via notebook cleanup or CLI)
          aws bedrock-agentcore-control delete-agent-runtime \
            --agent-runtime-id <runtime-id>

Step 2  Delete ECR repository
          aws ecr delete-repository \
            --repository-name langgraph-agent-repo \
            --force

Step 3  Delete Web Crawler Data Source (тЪая╕П orphaned тАФ not deleted by CFN stack)
          # Get data source ID first:
          aws bedrock-agent list-data-sources \
            --knowledge-base-id <PetCare2ndKnowledgeBaseId>
          # Then delete:
          aws bedrock-agent delete-data-source \
            --knowledge-base-id <PetCare2ndKnowledgeBaseId> \
            --data-source-id <WebCrawlerDataSourceId>

Step 4  Delete CloudFormation stack
          aws cloudformation delete-stack \
            --stack-name pet-store-knowledge-bases
          # Wait for DELETE_COMPLETE (~3-5 mins):
          aws cloudformation wait stack-delete-complete \
            --stack-name pet-store-knowledge-bases

Step 5  Empty and delete S3 knowledge data bucket
          aws s3 rm s3://<knowledge-bucket> --recursive
          aws s3 rb s3://<knowledge-bucket>

Step 6  Empty and delete S3 code staging bucket
          aws s3 rm s3://<code-bucket> --recursive
          aws s3 rb s3://<code-bucket>

Step 7  Delete Lambda backend functions (inventory + user management)

Step 8  Delete IAM SolutionAccessRole
```
